\section{Introduction}

This project studies circular trajectory tracking for a Boeing 747 model in planar flight. The goal is to build a compact workflow that combines control, nonlinear simulation, and state estimation inside one executable framework. The implemented architecture has three core components: an LQR controller for heading and bank regulation, RK4 numerical integration for nonlinear propagation, and a Gaussian Bayes/MAP estimator for estimation diagnostics.

The overall structure of the project is also aligned with the learning objectives of AAE6202, which emphasize optimization, computational solution of differential equations, and uncertainty-aware mathematical methods for aviation engineering applications \supercite{liu2026aae6202}.

LQR is a standard optimal control method for linear systems with quadratic costs and is widely used in aerospace inner-loop design because of its predictable tuning behavior and strong stability properties \supercite{anderson1990,lewis2012,astrom2008}. For discrete-time dynamics
\[
x_{k+1}=A_d x_k + B_d u_k,
\]
the infinite-horizon LQR minimizes
\[
J=\sum_{k=0}^{\infty}\left(x_k^\top Q x_k + u_k^\top R u_k\right),
\]
with \(Q \succeq 0\) and \(R \succ 0\), leading to linear feedback \(u_k=-Kx_k\). In this project, LQR is used to stabilize heading and bank-angle errors around a circular-tracking reference.

Aircraft motion remains nonlinear, so numerical integration is required. RK4 provides a practical balance between computational cost and accuracy and is commonly used for control-oriented flight simulation \supercite{hairer1993,butcher2016}. Compared with forward Euler, RK4 reduces truncation error significantly while remaining straightforward to implement.

The third component is Gaussian Bayesian estimation. Under Gaussian assumptions, a Bayes update is equivalent to a weighted MAP problem, linking estimation with uncertainty interpretation \supercite{kalman1960,simon2006,welch2006}. In the current code, the estimator is used for diagnostics rather than closed-loop feedback, but it still provides insight into innovation magnitude, covariance contraction, and posterior consistency.

More broadly, the present work is also related to the larger optimization literature. The quadratic-cost formulation behind LQR is naturally connected to classical linear and nonlinear programming \supercite{luenberger2008}, while the tuning of state and input penalties follows the logic of convex optimization and structured tradeoff design \supercite{boyd2004}. From the viewpoint of nonlinear state evolution, the trajectory-tracking problem can also be interpreted as a controlled dynamical system whose transient and asymptotic behavior is best understood in state-space terms \supercite{scheinerman2012,dilao2023}. In addition, although the current project is not formulated as a full uncertainty-aware optimization model, its estimator and sensitivity studies connect naturally to stochastic programming, robust optimization, and distributionally robust optimization \supercite{birge1997,haneveld2020,bental2009,bertsimas2011,blanchet2024,bomze2022}.

From a computational-methods perspective, the choice of RK4 is also aligned with standard engineering numerics references, which emphasize that higher-order integration is especially valuable when nonlinear coupling accumulates long-horizon propagation error \supercite{griffiths2020}.

The main contribution of this project is the integration of these three ideas into one consistent aerospace simulation workflow. The framework demonstrates how trajectory guidance, optimal regulation, nonlinear propagation, and stochastic state estimation can be combined in a concise course project.

\section{Problem Statement}

\subsection{Objective}

The objective is to design and evaluate a controller-estimator simulation system that drives a Boeing 747 model to track a circular reference trajectory in planar flight while maintaining a target speed.

\subsection{State, Input, and Reference}

The continuous state is
\[
\mathbf{x}=[x,\,y,\,\psi,\,\phi,\,v]^\top,
\]
where \(x\) and \(y\) are inertial coordinates, \(\psi\) is heading, \(\phi\) is bank angle, and \(v\) is airspeed.

The control input is
\[
\mathbf{u}=[\phi_{\mathrm{cmd}},\,T]^\top,
\]
where \(\phi_{\mathrm{cmd}}\) is the commanded bank angle and \(T\) is thrust.

The reference trajectory is a counterclockwise circle centered at \((0,0)\) with radius
\[
R=120{,}000\,\mathrm{m}.
\]
This baseline reference geometry is a project design choice defined in the simulation configuration and reused throughout the nominal and sensitivity experiments \supercite{aae6202code2026}.

\subsection{Performance Requirements}

The tracking system should satisfy the following requirements:
\begin{enumerate}
\item Reduce the signed cross-track error \(e_{ct}=R-r\) toward zero.
\item Keep heading and bank-angle behavior smooth and stable.
\item Maintain airspeed near \(v_{\mathrm{ref}}=210\,\mathrm{m/s}\), as specified in the baseline controller configuration \supercite{aae6202code2026}.
\item Respect actuator and physics limits on bank angle and thrust.
\end{enumerate}

\subsection{Constraints and Assumptions}

The problem is simplified by the following assumptions:
\begin{enumerate}
\item Flight is modeled in a 2D plane.
\item Aerodynamic drag is represented by a quasi-steady model.
\item Control is updated discretely with \(\Delta t=0.05\,\mathrm{s}\), while the state dynamics are propagated continuously through numerical integration; this baseline step size follows both the project implementation and the supplementary RK4 study setup \supercite{aae6202code2026,aae6202methodology2026,aae6202b747script2026}.
\item Measurement noise is Gaussian, and the estimator is used only for diagnostics in the current version.
\end{enumerate}

\subsection{Nonlinear B747 Planar Dynamics}

From \texttt{dynamics.py}, the planar model is
\[
\dot x = v\cos\psi,\qquad
\dot y = v\sin\psi,
\]
\[
\dot\psi = \frac{g\tan\phi}{v},\qquad
\dot\phi = \frac{\phi_{\mathrm{cmd}}-\phi}{\tau_\phi},\qquad
\dot v = \frac{T-D}{m}.
\]

The aerodynamic drag model is
\[
D=\frac12 \rho v^2 S C_D,\qquad
C_D=C_{D0}+k C_L^2,\qquad
C_L=\frac{2mg}{\rho v^2 S \cos\phi}.
\]
This particular planar B747 parameterization is cross-checked against the supplementary methodology note and the independent benchmark script, then implemented in \texttt{dynamics.py} \supercite{aae6202methodology2026,aae6202b747script2026,aae6202code2026}.

Key parameters from \texttt{B747Params} are:
\begin{enumerate}
\item \(m=333{,}400\,\mathrm{kg}\), \(S=541.2\,\mathrm{m}^2\)
\item \(C_{D0}=0.02\), \(k=0.045\)
\item \(\tau_\phi=2.0\,\mathrm{s}\)
\item \(|\phi|\le35^\circ\), \(T\in[0,\,1.1\times10^6]\,\mathrm{N}\)
\item \(v\in[120,\,320]\,\mathrm{m/s}\) with safety clipping
\end{enumerate}
The first three entries are inherited from the supplementary B747 model, whereas the saturation and clipping bounds are implementation choices adopted for numerical robustness in the current project \supercite{aae6202methodology2026,aae6202b747script2026,aae6202code2026}.

\subsection{RK4 Discretization}

For each simulation step, with constant input over \([t_k,t_{k+1}]\), RK4 evaluates
\[
k_1=f(x_k,u_k),\qquad
k_2=f\left(x_k+\frac{\Delta t}{2}k_1,u_k\right),
\]
\[
k_3=f\left(x_k+\frac{\Delta t}{2}k_2,u_k\right),\qquad
k_4=f\left(x_k+\Delta t\,k_3,u_k\right),
\]
and updates the state by
\[
x_{k+1}=x_k+\frac{\Delta t}{6}(k_1+2k_2+2k_3+k_4).
\]

The supplementary reference materials in \texttt{addicted/B747system.py} and \texttt{addicted/Methodology.pdf} are consistent with this RK4 modeling idea: they use the same five-dimensional state \([x,y,\psi,\phi,v]\), the same drag construction based on \(C_L\) and \(C_D\), and a direct comparison between RK4 and forward Euler under identical control inputs \supercite{aae6202methodology2026,aae6202b747script2026}. After cross-checking those materials, the present project keeps the standard RK4 weighting \(1,2,2,1\) in the final implementation and uses the comparison only as a verification study rather than as the main simulation engine \supercite{aae6202code2026}.

\subsection{Circular Tracking Controller}

From \texttt{controller.py}, the controller has three layers.

First, an outer-loop geometry-based guidance law computes current radius, circle tangent heading, and signed cross-track error. A bounded heading correction is added:
\[
\psi_{\mathrm{des}}=\psi_{\mathrm{tangent}}+\operatorname{clip}\left(\arctan\left(k_{ct}(-e_{ct})\right),\pm25^\circ\right).
\]

Second, a feedforward bank command for circular motion is generated from
\[
\phi_{ff}=\arctan\left(\frac{v_{\mathrm{ref}}^2}{gR}\right),
\]
with the correct sign for the direction of turn and with saturation at the bank-angle limit.

Third, an inner-loop LQR regulates heading and bank-angle errors,
\[
e_\psi=\psi-\psi_{\mathrm{des}},\qquad e_\phi=\phi-\phi_{ff}.
\]
Using the linearized error dynamics
\[
\dot e_\psi=\frac{g}{v_{\mathrm{ref}}}e_\phi,\qquad
\dot e_\phi=-\frac{1}{\tau_\phi}e_\phi+\frac{1}{\tau_\phi}u,
\]
where \(u=\Delta\phi_{\mathrm{cmd}}\), the code forms a discrete approximation
\[
A_d\approx I+\Delta tA,\qquad B_d\approx \Delta tB,
\]
solves the discrete Riccati equation, and applies
\[
\Delta\phi_{\mathrm{cmd}}=-K[e_\psi,\,e_\phi]^\top,\qquad
\phi_{\mathrm{cmd}}=\phi_{ff}+\Delta\phi_{\mathrm{cmd}}.
\]

The speed loop uses
\[
T = D_{\mathrm{guess}} + k_v(v_{\mathrm{ref}}-v),
\]
followed by clipping to thrust bounds.
The feedforward-turn formula comes from coordinated-turn kinematics, while the numerical values \(k_v=120000\), \(k_{ct}=1/3500\), and the \(\pm25^\circ\) heading-correction clip are baseline controller-tuning choices defined in \texttt{controller.py} \supercite{astrom2008,aae6202code2026}.

Default LQR weighting in \texttt{ControlConfig} is
\[
Q=\operatorname{diag}(30,\,10),\qquad R=[1].
\]
These weights are not taken from a handbook table; they are the baseline tuning used in the implementation and the later sensitivity study \supercite{aae6202code2026}.

\subsection{Gaussian Bayes/MAP Estimator}

From \texttt{estimation.py}, the estimator performs prediction, update, and MAP diagnostics.

Prediction:
\[
x_k^- = f_{RK4}(x_{k-1}^+,u_{k-1}),\qquad
P_k^- = P_{k-1}^+ + Q.
\]

Update:
\[
K_k=P_k^-(P_k^-+R)^{-1},
\]
\[
x_k^+=x_k^-+K_k(z_k-x_k^-),\qquad
P_k^+=(I-K_k)P_k^-.
\]

The MAP objective evaluated at the posterior state is
\[
\mathcal{L}_{MAP}
=\frac12(x_k^+-x_k^-)^\top(P_k^-)^{-1}(x_k^+-x_k^-)
+\frac12(z_k-x_k^+)^\top R^{-1}(z_k-x_k^+).
\]

The noise settings are:
\begin{enumerate}
\item Process standard deviation: \((2,\,2,\,0.2^\circ,\,0.2^\circ,\,0.5)\)
\item Measurement standard deviation: \((40,\,40,\,1^\circ,\,1^\circ,\,2)\)
\item Initial standard deviation: \((50,\,50,\,2^\circ,\,2^\circ,\,3)\)
\end{enumerate}
These covariance surrogates are project-level estimator settings encoded in \texttt{estimation.py} for diagnostic visualization rather than identified from flight-test data \supercite{aae6202code2026}.

\subsection{Simulation Workflow}

At each simulation step in \texttt{run.py}, the following sequence is executed:
\begin{enumerate}
\item Compute control action and control diagnostics.
\item Generate a noisy measurement.
\item Run the estimator update.
\item Propagate the true state with RK4.
\item Log telemetry for analysis and visualization.
\end{enumerate}

\subsection{Software Architecture and Data Flow}

From a software-engineering perspective, the project is organized as a compact closed-loop pipeline. The geometric reference is supplied to the controller, the controller generates bank-angle and thrust commands, the nonlinear B747 dynamics are propagated by RK4, and noisy measurements are processed by the Gaussian Bayes/MAP estimator. At the same time, telemetry signals such as tracking errors, timing, innovation norms, covariance traces, and state derivatives are collected for post-processing and report generation. This structure is intentionally modular: each mathematical part of the coursework corresponds directly to one code module.

\subsection{Cross-Check with Supplementary RK4/Euler Study}

The file \texttt{addicted/B747system.py} provides an independent numerical experiment for the same five-state B747 planar model. In that script, a fine-step RK4 solution with \(\Delta t_{\mathrm{ref}}=0.05\,\mathrm{s}\) is treated as reference, and both RK4 and Euler are then re-run at a coarser step size \(\Delta t_{\mathrm{test}}=0.5\,\mathrm{s}\) under a time-varying bank command. This is useful as an external consistency check for the numerical propagation part of the present project.

The script reports that at \(t=100\,\mathrm{s}\), RK4 reduces the final-state error norm from \(43.920\) (Euler) to \(2.730\), i.e. about \(16.09\times\) smaller. The average state error norm over the whole horizon is also reduced from \(28.479\) to \(2.250\), i.e. about \(12.65\times\) smaller. For translational states and airspeed, the RK4 advantage is particularly large: the final \(x\)- and \(y\)-position errors are about \(15.8\times\) and \(16.8\times\) smaller than Euler, and the final speed error is about \(128.6\times\) smaller.

Interestingly, the bank-angle error in that benchmark is not uniformly smaller for RK4 at every instant: the reported final \(\phi\)-error is \(0.459^\circ\) for RK4 versus \(0.298^\circ\) for Euler. This does not invalidate the overall conclusion, because the total state error and most state components still strongly favor RK4. The result mainly shows that, for a particular oscillatory input and coarse step size, a single component error may not monotonically reflect the global integration quality. Therefore, in this report, RK4 superiority is judged by the full state error norm and by trajectory-level closed-loop performance rather than by any one isolated state component.

These supplementary results support the design choice in the main project: using RK4 in \texttt{run.py} is justified not only theoretically but also empirically, since it provides substantially better nonlinear state propagation than Euler when the step size is not extremely small.

This matches the scalar statistics reported earlier: the mean prior error norm is \(8.380\), the mean posterior error norm is \(8.379\), and the posterior is better than the prior in about \(50.62\%\) of all steps. The phase portrait in Figure \ref{fig:error-phase-hist} also illustrates that the controller drives the joint error state toward the origin through a compact trajectory rather than through a wide oscillatory loop.

Figure \ref{fig:radius-sensitivity} compares four radii between \(80{,}000\,\mathrm{m}\) and \(200{,}000\,\mathrm{m}\), again using the sweep values specified in \texttt{extended\_reporting.py} \supercite{aae6202code2026}.

\subsection{Summary of Extended Experimental Evidence}

Taken together, the extended experiments paint a coherent picture. The nominal controller-estimator architecture is stable, computationally efficient, and physically well behaved. The main tracking results are robust to moderate time-step changes and to changes in reference curvature. Estimation quality degrades as sensor noise increases, but this degradation is isolated in the estimator because feedback still uses the true state. More aggressive LQR weights improve tracking but demand more control authority. Finally, both external and recreated benchmarks strongly confirm the numerical advantage of RK4 over forward Euler for this nonlinear flight-dynamics problem.

\section{Discussion}

The project demonstrates that a relatively simple guidance and regulation architecture can achieve precise circular tracking for a nonlinear planar aircraft model. The outer-loop geometric correction removes large radial offsets effectively, while the inner-loop LQR keeps heading and bank dynamics well behaved. This division of responsibility leads to a controller that is both interpretable and computationally light.

RK4 is important in this framework because the underlying aircraft model is nonlinear. The good steady-state performance indicates that numerical propagation error is not dominating the closed-loop response under the chosen time step. The supplementary benchmark in \texttt{addicted/B747system.py} further strengthens this point: even with a coarse integration step, RK4 remains much more reliable than Euler in overall state accuracy. The estimator results also show that the simulation environment supports uncertainty analysis, even though estimation is not yet used inside the feedback loop.

At the same time, the model remains simplified. The aircraft is limited to planar motion, the LQR design uses a first-order discretization, and the estimator is decoupled from the control action. These simplifications are acceptable for coursework demonstration, but they limit physical fidelity and robustness conclusions.

If the framework were extended in future work, one natural path would be to move from nominal optimization toward explicitly uncertainty-aware decision models. Stochastic-programming formulations could optimize expected performance across scenarios \supercite{birge1997,haneveld2020}; robust optimization could hedge against bounded but adverse parameter variations \supercite{bental2009,bertsimas2011}; and distributionally robust optimization could bridge engineering robustness with statistical ambiguity in deployment conditions \supercite{blanchet2024,bomze2022}. These references therefore provide a theoretical pathway for evolving the current simulation-and-diagnostics platform into a more general uncertainty-aware aerospace decision framework.

\section{Conclusion}

This project successfully demonstrates an integrated Boeing 747 circular-tracking simulation pipeline. LQR provides stable and accurate inner-loop regulation, RK4 supplies reliable nonlinear state propagation, and Gaussian Bayes/MAP estimation offers interpretable uncertainty diagnostics. Under the default configuration, the system reaches very small steady-state cross-track and speed errors while respecting actuator limits.

The main limitations are the planar flight assumption, simplified actuator and disturbance modeling, first-order discretization in LQR design, and the use of true state rather than estimated state for feedback. Future work should close the loop around estimated states, include wind and disturbance models, compare against alternative controllers such as MPC or nonlinear guidance laws, and extend the system to higher-fidelity aircraft dynamics.
